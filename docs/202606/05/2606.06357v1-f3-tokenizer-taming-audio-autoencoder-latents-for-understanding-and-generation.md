## 摘要
连续音频自编码器能够良好地重建波形，但其产生的潜变量在用于理解任务时往往结构较弱，而自监督音频编码器能捕捉语义却无法直接解码。这种不匹配使得一个必须同时支持理解与生成的单一音频分词器变得复杂。我们通过两个组件将连续自编码器潜变量适配到该场景：一个噪声正则化的自编码器瓶颈和一个潜变量侧表示编码器。瓶颈采用通道归一化和随机扰动而非基于KL的变分训练，为重建和自回归生成提供尺度可控的连续潜变量。表示编码器在冻结的自编码器潜变量上使用RQ-MTP和冻结的LLM监督进行训练。最终得到的分词器为理解任务提供高维表示，同时保留归一化的连续潜变量作为生成目标。

## Abstract
Continuous audio autoencoders reconstruct waveforms well but often produce latents with weak structure for understanding, while self-supervised audio encoders capture semantics but are not directly decodable. This mismatch complicates a single audio tokenizer that must support both understanding and generation. We adapt continuous autoencoder latents to this setting with two components: a noise-regularized autoencoder bottleneck and a latent-side representation encoder. The bottleneck uses channel normalization and stochastic perturbation instead of KL-based variational training, yielding scale-controlled continuous latents for reconstruction and autoregressive generation. The representation encoder is trained on frozen autoencoder latents with RQ-MTP and frozen-LLM supervision. The resulting tokenizer provides high-dimensional representations for understanding while preserving normalized continuous latents as generation targets

## 1. 核心问题与整体含义（研究动机和背景）

- **研究动机**：现有音频表示学习方法通常针对单一任务优化，难以同时兼顾高质量重建与语义理解。连续音频自编码器（如 SoundStream、EnCodec）虽能很好重建波形，但产生的潜变量缺乏结构化语义信息，不利于理解任务；自监督音频编码器（如 wav2vec 2.0、HuBERT）能捕获丰富语义，但无法直接解码为波形。这种“理解与生成”的不匹配阻碍了统一音频分词器的构建。
- **整体含义**：论文旨在设计一个**单一音频分词器**，既能输出高维表示用于理解（如 ASR、声纹识别），又能输出可解码的连续潜变量用于生成（如 TTS、文本到音频）。该工作聚焦于**分词器本身**的设计，而非完整的统一音频-语言模型。

## 2. 方法论：核心思想、关键技术细节

- **核心思想**：将连续自编码器潜变量作为“声学锚点”，在其上构建一个“表示编码器”，实现**双通路输出**：
  - 低维、归一化的连续潜变量 **z** → 用于重建和自回归生成。
  - 高维表示 **u** → 用于理解任务。

- **关键技术细节**：

  - **归一化自编码器 (Normalized Autoencoder)**：
    - 基于 SpectroStream 的 STFT 域骨干网络，替换瓶颈为**通道归一化 + 随机扰动**。
    - 通道归一化：对每个时间步的潜变量 z₀ 在通道维度做标准化：zₙ = (z₀ - μ_c)/(σ_c + ε_std)。
    - 随机扰动：训练时添加可控噪声 ẑ = zₙ + α·ε，α ~ U(0,γ), ε ~ N(0,I)。不采用 VAE 式 KL 正则化，而是通过归一化控制尺度，提升鲁棒性。
    - 解码器使用谱重建损失 + GAN 对抗损失（参考 DAC、SpectroStream）。

  - **潜变量侧表示编码器 (Latent-Side Representation Encoder)**：
    - 在冻结的自编码器潜变量 z 上训练因果表示编码器 Rη，输出表示 s（映射回解码器维度）和 u（用于自监督和语言监督）。
    - **RQ-MTP 自监督**：随机量化多令牌预测（类似 NEST-RQ），预测未来帧的量化目标，增强时序结构。
    - **冻结 LLM 监督**：将 u 下采样后投影为音频嵌入 e，输入冻结的 LLM，计算文本交叉熵损失 L_llm，对齐表示与文本。
    - **重建约束**：投影 s 通过自编码器解码器重建波形，强制保留声学信息。

  - **补丁级流头 (Patch-Wise Flow Head)**：
    - 用于生成：将连续潜变量 z 按补丁（patch）分组，每个补丁包含 P 帧。
    - 冻结 LLM 的音频对齐隐状态 q_t 条件化一个补丁级流匹配头（Flow Head），预测下一个连续潜变量补丁 a_t。
    - 训练目标为流匹配损失（参考 Flow Matching）。
  - **训练流水线**：
    - Stage 0：训练归一化自编码器（重建优化）。
    - Stage 1：冻结自编码器和 LLM，训练表示编码器、LLM 投影、补丁流头（包含 RQ-MTP、LLM 监督、流损失）。
    - Stage 2：任务特定训练（如 ASR 探针、TTS 生成器），分词器固定。

- **公式/算法流程**（文字说明）：
  - 自编码器：x → Eϕ → h → Bϕ → z₀ → 归一化 + 噪声 → ẑ → Dψ → ˆx (重建损失 L_ae)。
  - 表示编码器：z → Rη → s, u。s 返回解码器；u 用于 RQ-MTP 损失和 LLM 投影。
  - LLM 监督：u → Down → Pρ → e → Frozen LLM → 交叉熵损失 L_llm。
  - 生成头：q_t (来自 LLM) + 历史 z → 流头 → 预测潜变量补丁 a_t (流匹配损失)。

## 7. 优点

- **设计简洁统一**：不引入额外编码器、离散码本或多 token 流，仅通过瓶颈改进和表示编码器实现“一鱼两吃”。
- **归一化瓶颈创新**：避免 VAE 式 KL 正则化，使用通道归一化 + 随机扰动，保持潜变量尺度可控，训练稳定，且不损失解码能力。
- **表示编码器训练策略**：结合自监督（RQ-MTP）和语言监督（冻结 LLM），并强制重建约束，确保语义与声学信息的同步捕获。
- **生成感知的训练**：在分词器训练阶段引入补丁级流头，使潜空间对自回归生成更友好，提升下游生成训练效率。
- **实验全面**：覆盖重建、理解、生成三大维度，消融实验清晰验证各组件贡献。
