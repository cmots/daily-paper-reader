## 摘要
语音大语言模型缺乏一种原生的流式推理机制：它们的标签同步生成没有声学帧对齐，使得实时解码和话语结束检测变得困难。我们提出TRADE（换能器增强解码器），它通过一个共享音频编码器的换能器分支来增强多模态大语言模型，并直接使用大语言模型的隐藏状态作为预测网络——将帧同步声学对齐与大语言模型的语言推理能力耦合在一起。三种设计选择使系统准确、可流式处理且支持长格式语音：(1) 紧密耦合的双词汇表——从大语言模型词汇表派生的紧凑换能器词汇表，实现零成本分数融合；(2) 带有梯度停止的块同步流式训练，以离线等效的内存成本消除训练-推理不匹配；(3) 局部解码器音频注意力（LDAA），一种因果滑动窗口，将键值缓存内存限制在独立于话语长度的范围内。单个TRADE检查点支持在连续延迟操作点范围内进行离线和流式解码。TRADE在开放语音识别排行榜上实现了6.71%的平均词错误率，而使用960毫秒块大小的流式识别从同一检查点达到8.40%。在长格式语音上，它在TED-LIUM上获得3.64%的词错误率，在Earnings-22上获得10.88%的词错误率，无需外部分段。TRADE提供句尾标点时间戳，与声学语音活动检测结合时，使话语结束检测的F1分数比单独使用声学语音活动检测提高了+0.03。

## Abstract
Speech Large Language Models (Speech LLMs) lack a principled mechanism for streaming inference: their label-synchronous generation has no acoustic-frame alignment, making real-time decoding and end-of-utterance detection difficult. We propose TRADE TRansducer-Augmented DEcoder, which augments a multimodal LLM with a transducer branch that shares the audio encoder and uses the LLM's hidden states directly as the prediction network -- coupling frame-synchronous acoustic alignment with the LLM's linguistic reasoning. Three design choices make the system accurate, streamable, and long-form capable: (1)Tightly coupled dual vocabularies -- a compact transducer vocabulary derived from the LLM vocabulary, enabling zero-cost score fusion; (2)Chunk-synchronized streaming training with gradient stopping, eliminating the train-inference mismatch at offline-equivalent memory cost; and (3)Localized Decoder Audio Attention (LDAA), a causal sliding window that caps KV-cache memory independently of utterance length. A single TRADE checkpoint supports offline and streaming decoding across a continuous range of latency operating points. TRADE achieves 6.71% average WER on the Open ASR Leaderboard, while the streaming recognition with 960ms chunk size reaches 8.40% from the same checkpoint. On long-form speech, it obtains 3.64% WER on TED-LIUM and 10.88% on Earnings-22 without external segmentation. TRADE provides sentence-end punctuation timestamps that, when combined with acoustic voice activity detection (VAD), improve end-of-utterance detection by +0.03 F_1 over acoustic VAD alone.

## 1. 核心问题与整体含义（研究动机和背景）

- **核心问题**：现有语音大语言模型（Speech LLMs）采用标签同步（label-synchronous）的解码方式，即每步自回归生成一个 token，与声学帧之间缺乏显式对齐。这导致两个关键困难：
  - **流式推理**：模型无法确定何时消耗新的音频帧、何时输出 token，实时解码变得困难。
  - **话语结束检测（end-of-utterance detection）**：缺乏帧同步对齐的原生机制，无法可靠判断一句话是否结束。
- **背景**：现有流式方案分为两类：
  - **硬编码流式策略**：固定分块或 token 交错，由外部策略控制发射时机，缺乏对齐。
  - **辅助 CTC/transducer**：作为侧信号或特殊时间 token 提供对齐，但与 LLM 隐藏状态耦合松散。
  - **二者共同缺陷**：流式对齐与语言推理未能紧密集成，没有共享隐藏状态。
- **研究含义**：提出一种框架，在保持 LLM 语言推理能力的同时，赋予其帧同步的声学对齐机制，使单一模型能同时支持离线、流式、长格式语音识别，且能在连续延迟范围内灵活切换。

## 2. 方法论：核心思想、关键技术细节、公式或算法流程

### 核心思想
TRADE 通过一个共享的 Conformer 音频编码器，连接两条路径：LLM 路径（交叉熵损失）和 transducer 路径（RNN-T 损失）。利用 LLM 的隐藏状态直接作为 transducer 的预测网络（prediction network），实现帧同步对齐与语言推理的紧密耦合。

### 关键技术细节
1. **紧密耦合的双词汇表（Dual Vocabularies）**：
   - 从 LLM 全词汇表（约128K token）派生出紧凑的 transducer 词汇表（约20K token）。
   - 构造方法：保留原始 LLM token ID（零成本映射）；合并发音等价的表面形式（如 "OK"/"ok"）。
   - 推理时协作：transducer 发射 verbalized token 时，LLM 概率通过同音集合边缘化（Eq.2）与 transducer 分数融合（Eq.3），恢复大小写、标点等表面形式。空白步（blank）则查询 LLM 发射非 verbalized token（标点等）。
   - 公式：
     - 边缘化：$\log \tilde{p}_{LLM}(c) = \text{logsumexp}_{v \in H(c)} \log p_{LLM}(v)$
     - 融合选择：$\hat{c} = \arg\max_{c \neq \emptyset} [w \log p_{trans}(c) + (1-w) \log \tilde{p}_{LLM}(c)]$
2. **局部解码器音频注意力（LDAA）**：
   - 在 LLM 的音频注意力上施加因果滑动窗口，使 LLM 仅关注最近 $\leq (N_l+2) \cdot C$ 帧（$N_l$ 左上下文块数，$C$ 块大小）。
   - 好处：KV 缓存大小与话语长度无关，支持长格式语音；消除陈旧上下文对对齐的干扰。
   - 窗口默认左上下文5秒（大于99% acoustic support span 3.28秒）。
3. **块同步流式训练（Chunk-synchronized Training）**：
   - 在每个 chunk 边界，使用当前窗口音频和之前解码 token 重新预填充 LLM，得到预测状态 $s_u(t)$（Eq.5）。
   - **动态 chunk 训练**：每步从候选集随机采样 chunk 大小（包括全上下文），使单一检查点可部署于不同延迟点。
   - **梯度停止**：在流式训练中，transducer 反向传播时从 LLM 边界停止梯度，避免存储每个 chunk 的 LLM 激活，使峰值内存与离线训练相当。
4. **训练目标**：
   - 联合损失：$L_{total} = (1-\alpha) L_{ce} + \alpha L_{trans}$，默认 $\alpha=0.5$。
   - transducer 损失使用 k2 pruned RNNT 算法，仅保留高概率路径，使大词汇表 transducer 训练可行。
5. **解码流程（Algorithm 1）**：
   - 流式：音频逐块到达，每个 chunk 边界预填充 LLM，然后逐帧运行 joint network，根据空白/非空白决定发射 token，并融合 LLM 分数。
   - 离线视为 chunk 大小等于全长的特殊情况。

## 6. 主要结论与发现

1. **单一检查点支持多种模式**：TRADE 可在不修改架构的情况下，通过推理时选择 chunk 大小实现离线、流式（320ms~5120ms）操作，性能连续可调。
2. **准确性与流式能力并存**：
   - 离线：Open ASR Leaderboard 平均 WER 6.71%，优于 decoder-only LLM 基线（6.87%）。
   - 流式（960ms）：8.40%，仍有竞争力。
   - 延迟分析显示：640ms chunk 处于 WER-延迟曲线的膝点，推荐使用。
3. **长格式语音无需外部分段**：流式解码结合 LDAA 和 token 滑动窗口，实现单遍处理，TED-LIUM 3.64%，Earnings-22 10.88%，超过同期比较模型。
4. **端点检测性能提升**：将 transducer 的标点发射与 VAD 融合，F1 0.482，比单独 VAD 提高 0.031，平均检测延迟约 0.416s。
5. **融合权重鲁棒**：在 $w \in [0.1, 0.7]$ 范围内 WER 变化小于 0.06%，最优 $w=0.3$ 略好于两端，证明 transducer 和 LLM 提供互补信息。
6. **词汇大小选择**：20K 词汇在流式模式下显著优于 10K 和 15K，建议使用。

## 7. 优点

- **技术新颖性**：将 transducer 分支与 LLM 紧密耦合，LLM 隐藏状态直接作为预测网络，实现帧同步对齐，这是首个在该方向上深入工作的框架。
- **实用性强**：单一模型支持离线、流式、长格式，无需额外分割模型或后处理，大大简化部署。
- **训练-推理一致**：块同步训练配合动态 chunk，消除不匹配；梯度停止策略使内存开销可接受。
- **效率设计**：LDAA 限制 KV 缓存，token 滑动窗口防止长格式下 LLM 上下文膨胀。
- **词汇表设计巧妙**：紧凑 transducer 词汇表保留 LLM token ID，零成本映射；融合时通过同音边缘化恢复表面形式，无需外部语言模型后处理。
- **实验详尽**：包含多维度评估（精度、延迟、内存、端点检测），消融实验充分，可视化清晰（WER-AL 图、延迟指标图）。
- **开放性好**：论文提供大量附录细节（模型配置、词汇构造、训练数据、延迟分析），便于复现。

## 8. 不足与局限

1. **仅评估英语**：词汇表派生过程依赖英语发音规则，未测试其他语言。多语言能力尚未验证。
2. **流式精度退化**：流式与离线存在明显 WER 差距（如640ms 时 Leaderboard 平均 9.35% vs 6.71%），低延迟场景用户需接受这一权衡。
3. **端点检测实验有限**：仅在一个数据集（TED-LIUM，11个演讲）上测试，且参考边界来自自动分割器，非人工标注。F1=0.48 仍不高，在自发言语中效果未知。
4. **计算资源要求高**：需 16×H200 GPU 和约153K小时训练数据，对小团队复现有门槛。未报告不同 seed 或多次运行的稳定性。
5. **比较的局限性**：Open ASR Leaderboard 对比模型采用不同训练数据和规模，非严格公平。内部 decoder-only LLM 基线虽匹配，但未对比其他同类流式方法（如 Speech ReaLLM, BESTOW, Moshi 等）。
6. **长格式 WER 未与流式对比**：长格式结果只给出 5120ms chunk 模式，未报告同一模型在离线或更小 chunk 下的 WER，无法评估长格式下流式退化程度。
7. **词汇表构造的局限性**：合并同音形式假设英语发音规则单一，可能导致某些发音变体丢失（如方言）。同时，20K 词汇对罕见词仍会分裂成多个 BPE token，可能影响精度。
