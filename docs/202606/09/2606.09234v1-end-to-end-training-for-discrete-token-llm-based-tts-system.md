## 摘要
最近最先进的文本转语音系统通常采用级联管道，包括语音分词器、自回归大语言模型和基于扩散的流匹配模型，这些组件独立训练。在本文中，我们提出了一种全端到端优化框架，统一了语音分词器、LLM、FM模型以及额外奖励模型的训练。具体来说，我们首先使用来自FM重建、LLM下一个令牌预测和RM多识别任务的多任务目标联合优化分词器。这种联合训练鼓励离散语音令牌空间捕获更适合TTS的声学和语义显著信息。然后，我们通过FM和RM的下游重建和识别进一步优化LLM，减少推理时的不匹配，并引导LLM生成更优的结果。实验结果表明，我们的端到端框架始终优于级联基线。在Seed-TTS-Eval基准测试中，我们的系统实现了0.78%和1.56%的词错误率，使用0.6B参数的LLM和0.5B参数的FM模型取得了新的最先进结果。这些结果验证了整体端到端优化对于改进基于离散令牌的TTS系统（使用更简单的训练管道）至关重要。

## Abstract
Recent state-of-the-art (SOTA) text-to-speech (TTS) systems typically adopt a cascaded pipeline consisting of a speech tokenizer, an autoregressive large language model (LLM), and a diffusion based flow-matching (FM) model, with these components trained independently. In this paper, we propose a fully end-to-end (E2E) optimization framework that unifies the training of the speech tokenizer, LLM, FM model, and an additional reward model (RM). Specifically, we first jointly optimize the tokenizer using multi-task objectives derived from reconstruction for FM, next-token prediction for LLM, and multi recognition task for RM. This joint training encourages the discrete speech token space to capture acoustically and semantically salient information that is better tailored to TTS. We then further optimize the LLM using downstream reconstruction and recognition by FM and RM, which reduces inference-time mismatch and steers the LLM toward more preferred generations. Experimental results show that our E2E framework consistently outperforms cascaded baselines. On the Seed-TTS-Eval benchmark, our system achieves a word error rate (WER) of 0.78% and 1.56%, a new SOTA result with a 0.6B-parameter LLM and 0.5B-parameter FM model. These results validate that holistic E2E optimization is critical for improving discrete-token-based TTS systems with a much simpler training pipeline.

## 1. 核心问题与整体含义（研究动机和背景）
- **问题**：当前最先进的TTS系统通常采用**级联管道**（语音tokenizer → 自回归LLM → 流匹配FM模型），各组件独立训练。这导致三个关键缺陷：
  - tokenizer的训练目标（如ASR准确率、自编码保真度）与LLM的下一个token预测、FM的重建过程**未对齐**，限制了性能上限。
  - 离散语音token常包含冗余、纠缠且不可控的变化因素（如语义token也携带声学属性），给LLM预测带来歧义。
  - FM在训练时使用**真实token**，推理时使用LLM预测的**不完美token**，产生训练-测试分布不匹配；LLM仅最小化交叉熵，未考虑合成语音的感知质量。
- **动机**：已有工作尝试部分联合优化（如tokenizer+FM、LLM+FM），但缺乏整体端到端优化。本文提出**全端到端**框架，统一训练tokenizer、LLM、FM和奖励模型RM，以缓解组件间的不匹配。

## 2. 方法论
### 核心思想
- 将**一阶损失**（基于真实token）和**二阶损失**（基于LLM预测的token）结合，实现端到端梯度传播，使tokenizer学到更适配TTS的表示，同时使LLM生成更“重建友好”且“识别一致”的token序列。

### 关键技术细节
1. **模型组件**：
   - **语音tokenizer**：基于Conformer+FSQ（Finite Scalar Quantization），8维低秩码本，每维3个码，总码本大小6561。
   - **奖励模型RM**：共享Transformer编码器，带ASR（CTC）、SER（交叉熵）、SPK（余弦相似度）三个头。
   - **LLM**：采用Qwen3-0.6B，扩展词表加入6561个语音token和4个控制token。
   - **FM**：DiT架构，0.5B参数，预测速度场从噪声恢复语音。
2. **一阶损失L1**（第一阶段）：
   - 输入真实token序列 \( c_{1:T}, q_{1:T} \)。
   - FM重建损失 \( L_{FM} = \|FM(q, \mu) - (x - n)\| \) 。
   - LLM下一个token预测损失 \( L_{LM} = \sum CE(c_t, p(c_t|c_{<t}, text)) \)。
   - RM多任务损失 \( L_{RM} = L_{ASR} + L_{SER} + L_{SPK} \)。
   - 联合优化：\( L_1 = \alpha L_{LM} + \beta L_{RM} + \gamma L_{FM} \)，α=0.1, β=1.0, γ=1.0。
3. **二阶损失L2**（第三阶段）：
   - 用Gumbel-Softmax或LLM隐藏状态 \( h^{LM} \) 生成预测token序列 \( q' \)。
   - 计算重建损失 \( L_{LFM} \) 和识别损失 \( L_{LRM} \)（ASR+SPK+SER）。
   - 梯度从FM和RM反向传播至LLM，引导LLM输出更符合下游需求。
4. **三阶段训练流程**：
   - **Stage 1**：所有参数可训练，使用L1损失，优化tokenizer、LLM、FM、RM。
   - **Stage 2**：冻结tokenizer，分别微调RM、FM和LLM（可引入不同数据子集，如含噪声数据增强RM）。
   - **Stage 3**：冻结RM，使用L2联合优化LLM和FM。

### 理论分析
- 将tokenizer训练视为**信源编码**问题：最大化token熵 \(H(X_n)\) 等价于最小化条件熵 \(H(Y_n|X_n)\)（重建不确定性）。FM损失直接优化此项。
- LLM预测需建模**记忆性**，通过一阶马尔可夫假设，最大化互信息 \(I(X_{n+1}; X_n)\) 可减少预测不确定性，LLM损失直接优化此互信息。
