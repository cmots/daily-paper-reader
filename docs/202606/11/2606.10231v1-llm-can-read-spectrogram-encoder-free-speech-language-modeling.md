## 摘要
近期基于语音的大语言模型（Speech-LLMs）依赖预训练的语音编码器将音频转换为LLM可消费的语义丰富表示。相反，在本工作中，我们探索：LLM能否直接学习读取Mel语谱图，而无需专门的语音编码器？我们提出Mel-LLM，一种无编码器的语音-LLM，通过线性投影将经过轻度预处理的Mel语谱图块直接输入LLM，使LLM完全通过自身参数学习语音-文本对齐。我们在自动语音识别（ASR）和文本到语音（TTS）任务上进行了广泛实验。对于ASR，我们在OpenASR排行榜公开集和生产级缩放实验上评估，表明无编码器方案性能具有竞争力，与使用编码器初始化的方案相比仅有有限退化。我们发现当数据有限时，从多模态检查点（Phi-4-MM）进行初始化对于保持性能至关重要。我们还通过消融研究揭示了哪些LLM层与语音编码相关性较低。对于TTS，我们展示了基于下一个token VAE方法的初步结果。虽然TTS性能尚不理想，但这些结果确立了用于自回归语音-文本建模的完全统一无编码器架构的可行性。

## Abstract
Recent speech-aware large language models (Speech-LLMs) rely on a pre-trained speech encoder to convert audio into semantic-rich representations consumable by LLM. In this work, instead, we explore: can an LLM learn to read Mel spectrogram directly without a dedicated speech encoder? We propose Mel-LLM, an encoder-free Speech-LLM that feeds lightly pre-processed Mel spectrogram patches directly into the LLM through a linear projection, allowing the LLM to learn speech-text alignment purely through its own parameters. We conduct extensive experiments on both automatic speech recognition (ASR) and text-to-speech (TTS) tasks. For ASR, we evaluate on the OpenASR leaderboard public sets and production-level scaling experiments, demonstrating that the encoder-free solution achieves competitive performance with only limited degradation compared to encoder-initialized counterparts. We find that when data is limited, initialization from a multimodal checkpoint (Phi-4-MM) is crucial for maintaining performance. We also present ablation studies revealing which LLM layers are less relevant to speech encoding. For TTS, we show preliminary results with a next-token VAE approach. While TTS performance is not yet optimal, these results establish the feasibility of a fully unified encoder-free architecture for autoregressive speech-text modeling.

## 1. 核心问题与整体含义（研究动机和背景）

- **研究动机**：当前主流的语音大语言模型（Speech-LLMs）普遍采用“预训练语音编码器 → 模态投影器 → 大语言模型”的三阶段架构（如 Whisper、Conformer 等编码器）。这种方案存在三个核心局限：① 编码器参数量大（如 Whisper-large 超 6 亿参数），增加计算开销；② 编码器学习到的表示可能与 LLM 内部处理不匹配，造成表示鸿沟；③ 编码器成为信息流瓶颈，LLM 只能访问压缩后的表示。
- **背景启发**：近期视觉-语言模型（如 Fuyu、Tuna-2）已证明 LLM 可以直接处理原始像素块而无需视觉编码器，暗示大语言模型有足够容量在内部学习模态专用处理能力。
- **核心问题**：**大语言模型能否直接读取 Mel 语谱图（Mel spectrogram），而不依赖任何专用语音编码器？**
- **整体含义**：若可行，则有望简化 Speech-LLM 架构，实现真正的端到端统一语音-文本建模，兼顾理解（ASR）与生成（TTS）。

## 2. 论文提出的方法论

### 2.1 核心思想：Mel-LLM

- 提出 **Mel-LLM**，一种“无编码器”语音-LLM 架构，完全移除传统的 Transformer/Conformer 编码器（仅保留用于下采样的卷积层）。
- 将 Mel 语谱图切分后，通过线性投影直接送入 LLM 的词嵌入空间，让 LLM 自身参数学习语音-文本对齐。

### 2.2 ASR（语音理解）路径

1. **特征提取**：80 维 log-Mel 语谱图。
2. **MVN 归一化**：基于训练集预计算统计量进行均值和方差归一化。
3. **轻量卷积（可选）**：用于时间维度下采样（ASR 中 r=8，即 12.5Hz 帧率）。
4. **线性投影**：单层线性层将特征映射到 LLM 隐藏维度（3072），投影权重随机初始化。
5. **LLM 处理**：投影后的语音嵌入与文本提示嵌入拼接，送入 Phi-4-MM LLM（32 层，隐藏维度 3072），使用 LoRA 适配（rank=320, α=640）。
6. **自回归文本生成**：LLM 输出转录文本 token。

### 2.3 TTS（语音生成）路径

- 采用 **下一个 token 预测** 自回归生成连续 Mel 帧。
- **VAE 解码器**：从 LLM 隐藏状态解码 Mel 语谱图，包括：
  - 线性层输出均值 μ 和对数方差 logσ²
  - 重参数化采样 z = μ + σ·ε
  - 残差 MLP（3 层线性层 + tanh + dropout）得到预测 Mel
  - 5 层 Conv1D 残差 PostNet 进行细化
- **停止预测**：线性层预测音频结束标志（BCE 损失）。
- **训练损失**：L1+MSE 重构损失 + KL 散度（权重 0.05）+ 停止损失（权重 1.0）+ 频谱通量损失（权重 0.5，鼓励时间平滑性）。

### 2.4 关键差异

- **无编码器**：完全不用预训练语音编码器。
- **初始化依赖**：数据有限时需从多模态检查点 Phi-4-MM 初始化 LoRA 权重；随机初始化会导致性能显著下降。

## 6. 论文的主要结论与发现

1. **无编码器 ASR 可行**：Mel-LLM 在 OpenASR 公开集上平均 WER 7.12%，仅比随机初始化的编码器版本（6.97%）差 0.15%，比 Phi-4-MM 全微调（5.61%）高 1.51%。
2. **多模态初始化至关重要**：数据有限时，Phi-4-MM 初始化比随机初始化 WER 低 0.32%（7.12% vs 7.44%），且 TTS 中随机初始化完全无法产生可听输出。
3. **数据缩放可弥合差距**：使用 10× 内部数据后，无编码器方案与编码器基线的相对差距从 +11.3% 缩小到 +3.8%。
4. **低层 LLM 承担语音编码角色**：冻结 LLM 前半层（L0-23）导致性能显著下降，而冻结高层（L24-31）影响很小，说明低层学习隐式语音编码，高层保留语言推理能力。
5. **下采样率需权衡**：12.5Hz 帧率（8× 下采样）在 ASR 中达到最佳速度-质量权衡（1.57× 训练加速，WER 7.12%）。
6. **TTS 初步可行**：采用 VAE 解码器可实现连续 Mel 自回归生成，但质量尚有优化空间（WER 11-18%，UTMOS 约 3.1-3.3）。

## 7. 优点

- **架构创新**：首次在 Speech-LLM 中彻底移除编码器，实现真正的“编码器无关”建模，简化系统设计。
- **统一框架**：同一 LLM 骨架可同时用于 ASR 和 TTS，为未来联合理解和生成奠定基础。
- **实验严谨**：提供了详细的消融研究（下采样率、层角色、初始化影响），并与有编码器版本进行逐层对比，结论可信。
- **实用意义**：在数据充足时无编码器方案性能逼近有编码器，且训练速度更快（1.57×），具有实际部署价值。
- **公开性**：ASR 训练完全使用公开数据集，便于复现验证。

## 8. 不足与局限

- **TTS 探索尚浅**：TTS 仅使用 Libriheavy 单一数据集，未与现有 TTS 系统（如 MELLE 原文效果、VITS 等）进行量化对比，WER 和 UTMOS 绝对数值偏低，远未达到实用水平。
- **未做联合训练**：ASR 和 TTS 实验分开进行，未统一在同一模型中实现双向语音-文本变换，作者也承认这是未来方向。
