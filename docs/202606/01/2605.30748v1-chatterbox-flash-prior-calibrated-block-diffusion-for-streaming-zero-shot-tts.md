## 摘要
我们提出了Chatterbox-Flash，这是一种通过将预训练的自回归TTS解码器微调为块扩散解码器而获得的零样本文本转语音模型，能够在每个块内并行生成令牌，同时保留逐块流式传输。我们发现，将主流的块扩散解码直接迁移到离散语音令牌会降低质量，因为长尾令牌分布会将并行位置选择偏向少数高频令牌。为了在不修改架构的情况下缓解这一问题，我们引入了两种推理时技术：先验校准评分（减去块级边际令牌分布）和早期解码调度（基于校准置信度自适应终止迭代）。在标准零样本TTS基准测试中，Chatterbox-Flash达到了与强自回归和非自回归基线相当的高保真合成，同时支持流式推理，其首包时间与流式AR系统相当，且实时因子显著更低。代码和音频样本可在https://github.com/resemble-ai/chatterbox-flash获取。

## Abstract
We present Chatterbox-Flash, a zero-shot text-to-speech model obtained by fine-tuning a pretrained autoregressive TTS decoder into a block-diffusion decoder, enabling parallel token generation within each block while retaining block-by-block streaming. We find that naively transferring mainstream block-diffusion decoding to discrete speech tokens degrades quality, as a long-tail token distribution biases parallel position selection toward a few high-frequency tokens. To mitigate this without architectural modification, we introduce two inference-time techniques: prior-calibrated scoring, which subtracts the block-level marginal token distribution, and an early-decoding schedule, which adaptively terminates iteration based on calibrated confidence. On standard zero-shot TTS benchmarks, Chatterbox-Flash attains high-fidelity synthesis comparable to strong autoregressive and non-autoregressive baselines, while supporting streaming inference with time-to-first-packet on par with streaming AR systems and substantially lower real-time factor. Code and audio samples are available at https://github.com/resemble-ai/chatterbox-flash.

## 1. 核心问题与整体含义（研究动机和背景）
- **问题**：零样本文本转语音（TTS）中，自回归（AR）模型推理速度慢（令牌顺序生成），而非自回归（NAR）模型虽然速度快，但通常不支持流式输出（需要整句生成）。现有方法无法同时兼顾高质量与流式低延迟。
- **背景**：扩散语言模型（DLM）已在文本生成中实现并行解码，但直接迁移到离散语音令牌时存在问题——语音令牌分布呈长尾（如静音令牌占比极高），导致平行位置选择偏向高频无信息令牌，产生模糊上下文，降低合成质量。此外，逐块解码（Block Diffusion）的局部窗口限制使排序更脆弱。
- **整体含义**：本文提出Chatterbox-Flash，通过将预训练AR解码器微调为块扩散解码器，并在推理时引入先验校准与早期解码，实现兼具高质量和低成本流式推理的零样本TTS模型。

## 2. 方法论：核心思想、关键技术细节

### 核心思想
- 将自回归TTS解码器（Chatterbox-TTS的Llama-style Transformer）改造为**块扩散解码器**：对序列进行分块，块内并行预测掩码令牌，块间以因果方式流式生成。
- 针对离散语音令牌的长尾分布，提出**先验校准评分（PMI）**和**早期解码调度**，无需修改架构即可缓解质量退化。

### 关键技术细节
- **模型架构**：基于Chatterbox-TTS两阶段流水线，仅替换Stage 1的解码器。保留原架构，将训练目标从自回归交叉熵改为**掩码去噪**。编码器保持不变。
- **训练**：
  - **混合注意力**：条件序列（说话人嵌入、文本、提示语音）使用因果注意力；每个语音块内使用双向注意力；跨块因果。保持与预训练骨干一致的单调对齐。
  - **互补掩码**：每次训练随机采样掩码概率，并同时使用掩码和补掩码两个视图监督每个位置。
  - **令牌移动损失（Token-Shift Loss）**：从下一令牌位置预测当前掩码令牌，保留自回归接口但允许块内双向上下文。
- **推理**：
  - **块自回归解码**：逐块提交；已提交块作为干净上下文缓存。
  - **先验校准评分**：使用点互信息（PMI）分数：`s_i = log p_i(x̂_i) - log p̄(x̂_i)`，其中`p̄`为无条件块先验（对全掩码块且条件置零的一次前向计算得到，可缓存）。这减去了边际令牌概率，避免高频无信息令牌被优先选择。
  - **早期解码调度**：基于PMI分数设定置信度阈值`θ_k`，自适应决定每步提交多少位置。结合时间偏移（TS）调度，用`max(f_k, g_k)`确定新增提交数，当α大时平均步数减少。
  - **分类器无关引导（CFG）**：条件与无条件分支结合，但PMI评分仅使用条件分支。

## 6. 主要结论与发现
1. **质量**：Chatterbox-Flash（PMI, α=0）在LibriSpeech-PC和Seed-TTS上WER分别为1.67%和1.96%，UTMOS分别为4.29和4.09，与最强AR/NAR基线持平或略优。
2. **流式效率**：首个令牌延迟（TTFP）118ms，与流式AR系统Qwen3-TTS相当；实时因子（RTF）0.107，比Qwen3-TTS最低变体（0.253）快约2.7倍。
3. **早期解码**：PMI+ED（α=0.5）可节省约20%步数，质量几乎无损失；α=1.0可节省约41%步数，WER小幅增加（+0.6）。
4. **PMI贡献**：在饱和基准（WER 1.7~2.0）上，PMI与TS调度基线质量等价，其优势在于提供校准置信度用于早期解码；在硬样本（EmergentTTS-Eval）上，PMI降低WER约10.6%（从38.52到34.42）。
5. **人类评估**：自然度（NMOS）3.91 vs ElevenLabs v3的4.04，但低分比率更低；说话人相似度（SMOS）4.56 vs 3.50，更好保留参考说话人特征。

## 7. 优点
- **方法创新**：首次将块扩散与流式结合应用于离散语音令牌；先验校准评分轻量有效，无需修改架构或额外前向。
- **推理效率**：通过CUDA Graph、paged KV缓存、混合注意力等优化，实现了非常低的RTF（0.076~0.107），支持生产级流式服务。
- **训练策略**：利用互补掩码和令牌移动损失，保持与预训练自回归骨干兼容，使微调稳定。
- **实验全面**：涵盖质量、效率、消融、人类评估，对比多种基线，分析清晰（如质量随步数/块大小的变化曲线）。
- **开放资源**：提供代码和音频样本。

## 8. 不足与局限
- **数据规模**：仅70k小时英语数据（多数据源但公共部分占比有限），远小于OmniVoice的581k小时多语言数据，可能限制泛化性和多语言能力。
- **大块尺寸不稳定**：块大小D≥128时训练崩溃（即使使用PMI），而全序列公式（如OmniVoice）保持稳定。作者尝试了几种变体（完全因果、块大小退火等）但均未成功，留下并行性差距问题。
- **PMI的优势有限**：在饱和基准（WER低的读语料）上，PMI与简单时间偏移调度（TS）质量等价。PMI的主要作用是提供校准置信度以支持早期解码，而非直接提升质量。在更紧的预算或域外参考上是否仍有帮助未验证。
- **伪流式对比缺失**：未与采用伪流式改编的OmniVoice（将文本分成小块处理）进行公平对比，因此流式优势是否来自块扩散本身还是训练对齐尚不清楚。
- **语言限制**：仅英语，未评估其他语言零样本性能。
- **失败模式**：当块大小从D=24开始，WER显著增加（LibriSpeech-PC上从1.61至2.38），说明并行承诺能力受限。
